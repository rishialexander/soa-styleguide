Overview

This is the style guide for the forthcoming School of Art + Art History + Design website. It provides tools, assets and guidance to streamline the workflow between designers and developers, and maintain a consistency in style across the entire site.

The style guide was last updated on December 29, 2015. For more information, contact Rishi Agarwal (rishialexander@gmail.com).

Typography

The Art + Art History + Design website is set entirely in Karla. To maximize readability, aim for 8-12 words per line when possible.

Components

Header

Desktop

The header connects the Art + Art History + Design departments with the greater University of Washington. The gray links go to various external pages, and the logo goes to the University of Washington homepage.

Mobile

On mobile devices the gray shortcut links are removed and replaced with a hamburger button ☰ that opens the navigation dropdown menu.

Primary navigation

Desktop

The navigation menu has four top-level items: one for each of the departments and a fourth for shared facilities. Hover over a top-level item to reveal all of the remaining levels in that category at once. The menu supports a maximum of three levels.

Mobile

On mobile, press the hamburger button ☰ in the header to reveal the navigation menu with a sliding animation. When the menu opens, only the top-level items are visible. Tap any top-level item to reveal all of the remaining levels in that category, in a similar fashion to the desktop experience. Plus symbols +, minus symbols –, and horizontal rules help to indicate when a menu is expanded. The hamburger button turns into an × to dismiss the menu.

Sidebars

Desktop

The sidebar makes it easy to find information about the current page, or access related pages. The topmost sidebar section contains links to the parent page, the parent page's siblings, and the current page's siblings. The sidebar section below it contains important information that is directly related to the current article.

Mobile

On mobile the entire page collapses to a single column. The sidebar appears directly below the main article. The line height for the links is increased to accommodate finger taps.

Forms

Footer

Desktop

The footer provides important information across four sections. (1) Contact information including the address, phone number and map to the school. (2) Links to navigate to the most important pages within the site. (3) Social links to stay connected or donate to the school. And (4) Links to legal and administrator pages. The background color is dark gray to blend into the site background.

Mobile

On mobile, the footer is collapsed into a single column. All of the content remains the same.

Layout

Desktop

This is the structure for a typical page featuring an article.

  • The article has a maximum width to prevent the line lengths from becoming too long.
  • The sidebar has a smaller maximum width. It's aligned to the top edge of the article and sticks to the right edge of the article.
  • The background image is a curated photo of an artwork and fills the entire page up to the fold. Below the fold, the background is a dark gray fill. The article and sidebar peek above the page fold and sit on top of the background image.
  • All of the content is aligned to the same left margin regardless of screen size. This includes the header, the logo, the article, and the footer.

Tablet

On tablet, the content is almost identical but there are significant visual changes to accommodate the smaller screen size.

  • The spacing inside and outside of each column is reduced to fit both columns on a tablet.
  • The links in the header are replaced with a hamburger button ☰ for the mobile navigation menu.
  • The logo is smaller than on desktop.
  • The footer is collapsed into two columns.

Mobile

On mobile, there are additional visual changes to accommodate the smallest screen sizes.

  • All of the content is collapsed to a single column that spans the width of the screen.
  • The links in the header are replaced with a hamburger button ☰ for the mobile navigation menu.
  • The logo is smaller than on desktop.
  • The background image is shrinks with the width of the page, but it has a minimum size so that it crops instead of filling the page.
  • The sidebar is positioned below the article.
